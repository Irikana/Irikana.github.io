提示框左侧竖线 Bug 修复

- 修复 .function-box-blue 和 .quote-box-grey 的双重左侧竖线问题

- 根因：两个组件同时使用了 CSS border-left 和 ::before 伪元素绘制两条叠加的竖线，导致放大后不对齐、手机端出现半黑线条

- 修复方案：移除 ::before 伪元素，统一仅使用 border-left 作为唯一竖线绘制机制

- 同步更新视觉组件标准文档（visual-components.md），移除过时的 ::before 定义

- 涉及文件：

  • css/style.css — 删除 .function-box-blue::before 和 .quote-box-grey::before，统一 border-left 宽度

知识馆变更（续）

知识馆侧边栏主题适配修复

- 修复知识馆侧边栏在「系统暗色 + 手动浅色」场景下背景仍为黑色的问题

- 根因：知识馆页面仅有 @media (prefers-color-scheme: dark) 和 .force-dark-mode 暗色规则，缺少 .force-light-mode 亮色覆盖规则。当系统为暗色模式时，即使手动选择浅色模式，媒体查询的暗色 CSS 仍然生效

- 修复方案：在全部 4 个知识馆页面添加完整的 .force-light-mode {...} 规则块，强制恢复所有组件的浅色外观（侧边栏、导航项、内容区、卡片、页脚等）

- 涉及文件：

  • knowledge-hall/index.html — 新增 .force-light-mode 规则块

  • knowledge-hall/categories/phenomenon.html — 新增 .force-light-mode 规则块

  • knowledge-hall/categories/recallable.html — 新增 .force-light-mode 规则块

  • knowledge-hall/categories/traceable.html — 新增 .force-light-mode 规则块
