# 牧羊人图书馆开发规范 (Shepherd's Library Development Standards)

## 🎯 技能用途

此技能封装了牧羊人图书馆（Shepherd's Library）项目的完整开发规范。**每次修改本项目时必须遵循以下规则**。

## ⚠️ 开发前必做：检查更新日志

### 强制流程
1. **查看最新日志**: 在开始任何代码修改前，必须先检查 `updateLog/` 目录中最新的更新日志
2. **确认版本号**: 确认当前版本号（当前: **alpha-006**）
3. **审查变更内容**: 了解最近的变更，避免冲突
4. **补充缺失记录**: 如发现未记录的变更，先补充日志再开发

## 🔒 核心原则（不可违反）

### 1. 文字与图片内容保护
- ✅ **保留原话**: 不要随意修改文字图片内容
- ✅ **保护现有内容**: 所有页面中的原有文字、链接、图片路径、JavaScript 逻辑均不得擅自修改
- ✅ **仅在明确要求时修改**: 仅当用户明确要求时方可改动文字或图片
- ✅ **使用提示词原话**: 用户需求描述中的具体文本可直接用于实现

### 2. 设计确认机制
遇到以下情况**必须主动向用户确认**:
- 需求存在多种合理实现方式
- 涉及视觉风格选择
- 用户意图不明确
- 需要删除或大幅重构现有功能

### 3. 红色警告框使用原则（严格限制）

#### ❌ 禁止使用的场景（改用蓝色信息框 `.function-box-blue` 或灰色引用框 `.quote-box-grey`）
- 创建日期 / 版本信息
- 参考性说明 / 引用来源说明
- 页面状态通知（如"尚未开发"）

#### ✅ 允许使用的场景（需作者明确许可）
- Alpha/Beta 阶段的重要建设状态提醒
- 内容时效性警示
- 需要读者特别注意的约束条件

**默认优先级**: 蓝色信息框 > 灰色引用框 > 红色警告框

## 📝 版本控制规范

### 版本号格式
- **方案B**: `alpha-{自定义编号}`（如 `alpha-001`, `alpha-004`）
- 编号由作者自主控制，**不绑定日期**
- 当前版本: **alpha-006**

### 更新日志规则
- **净变更保存**: 每次开发的最终变更都保存到更新日志
- **日志位置**: `updateLog/updateLog_alpha-{编号}.html`
- **分区域记录**: 主馆变更 / 知识馆变更 / 规范变更 各自独立列出
- **净变更定义**: 对比上一次版本的最终更改（不含中间修正过程）

## 💻 代码规范

### HTML 结构要求
```html
<!-- ✅ 正确示例 -->
<header>
  <nav>
    <main>
      <section>
        <article>
          <img src="./image/logo.png" alt="语义化描述">
```

- 使用语义化标签: `<header>` `<nav>` `<main>` `<footer>` `<section>` `<article>`
- 统一 **2空格缩进**
- 为 `img` 添加语义化 `alt` 属性
- 移除空标签和垃圾类名（如 `c28806` 等）

### CSS 规范
- 使用 CSS 变量 (`:root`) 管理设计令牌（颜色、间距等）
- 保持响应式设计兼容（桌面端 / 平板 / 手机）
- 支持暗色模式: `@media (prefers-color-scheme: dark)`

## 🎨 页面布局规范

### 主页板块结构（三级大板块）
1. **新闻** — 轮播展示最新动态
2. **前情提要** — 包含子板块：
   - 图书馆入门
   - 图书馆规则
   - 图书馆功能
3. **入口** — 图书馆入口

知识馆以浮窗/浮动按钮形式出现，**不作为大板块**占据主内容区。

### 知识馆风格规范
- 整洁、简约的设计
- 文字醒目，适当增大字体
- 取消顶部导航栏（nav），改用侧边栏导航
- 侧边栏包含：三种知识分类入口、回到图书馆主页、进入图书馆入口

### 新闻卡片创建规范
当用户提供时间、标题和内容要求创建新闻时，遵循以下标准：

**两种新闻类型模板：**
1. **文字新闻** — 使用 `news-card-text-only`，hint 固定为"点击此处了解更多"
2. **海报新闻** — 使用 `news-card-content` + `news-card-image`，hint 固定为"点击海报了解更多"

**关键规则：**
- `data-date` 格式为 `YYYY-MM-DD`（用于轮播降序排序）
- **hint 不得放入正文内容**，正文放在链接目标页面中
- 中英文主页新闻需同步添加
- 新卡片插入到 `#carousel-track` 最前面
